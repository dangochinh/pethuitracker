# Specification: Internationalization (i18n)

## Overview
This specification details the behavioral requirements for the newly introduced multi-language support feature, transitioning Pe Thui Tracker from a single-locale Vietnamese experience to a dual-locale (Vi/En) system.

## Behavior Driven Development (BDD) Requirements

### Requirement: Language Selection & Persistence
- **GIVEN** a new user visits the application for the first time
- **THEN** the application defaults to Vietnamese (`vi`).
- **GIVEN** a user opens the settings/info modal
- **WHEN** they toggle the language selector block to English (`en`)
- **THEN** the UI text immediately updates to English
- **AND** the preference is saved in localStorage (`pe_thui_locale`) and synced to a cookie for SSR consistency.
- **AND** on subsequent application reloads, the user interface boots in English.

### Requirement: Data Model Formatting
- **GIVEN** static dictionary dependencies like vaccinations (`vaccines.js`) and physical growth milestones (`skills.js`)
- **WHEN** they are rendered into the UI (e.g., Timeline or Lists)
- **THEN** the view logic correctly dereferences the ID to the underlying translation dictionary `t(vaccine.nameKey)` instead of displaying hardcoded model parameters.

### Requirement: Profile Backend Synchronization
- **GIVEN** an existing child profile
- **WHEN** the parent explicitly saves their profile form
- **THEN** the active locale preference (`en` or `vi`) is captured and persisted to the underlying Google Sheets Database into the `A1:B5` Profile Block.

### Requirement: Cron Job Notification Targeting
- **GIVEN** the background cron job scans the Google Sheet database each morning
- **WHEN** evaluating a profile with a 3-day notification milestone
- **AND** that profile has the `Locale: en` stored
- **THEN** the Web Push broadcast and Telegram Bot webhook deliver the reminder specifically using the English templates instead of the default Vietnamese.
